# Архитектура UnityCode Network (Версия 1.0)

## 1. Обзор
UnityCode Network — децентрализованная сеть нейротворчества, построенная вокруг личных узлов участников. Каждый узел является форком этого репозитория и содержит артефакты синтеза человеческих мыслей с ИИ-интерпретацией. Портал [unitycode.space](https://unitycode.space) выступает точкой входа, а GitHub служит распределённым хранилищем и социальным графом.

## 2. Философское ядро
Мироздание познаёт себя через нас. Творчество — это создание связей. Нейротворчество — синтез потоков человеческих мыслей и их интерпретация ИИ с целью интеграции разрывов и усиления общей связности (индекса Φ).

## 3. Компоненты системы
### 3.1 Портал (unitycode.space)
- Входная точка: чёрный фон, минимум элементов.
- Модуль «Петля»: ввод сырого текста (хаоса) и получение интерпретации от ИИ.
- Модуль «Сеть»: визуализация активных узлов (форков) и сигналов связи между ними.
- Авторизация через GitHub OAuth для выполнения действий от имени пользователя.

### 3.2 Личный узел (форк)
- Создаётся автоматически при первом «вплетении» интерпретации.
- Структура: `traktat/` (сшитые тексты), `signals/` (файлы связей), `config.yml` (настройки).
- Узел остаётся полностью под контролем участника.

### 3.3 ИИ-интерпретатор
- Работает по системному промпту 2.0.
- Анализирует входной сигнал, находит разрывы, предлагает интеграции.
- Не имитирует эмоции, не даёт терапевтических советов.

### 3.4 Коллективный трактат
- Формируется автоматически путём агрегации публичных `traktat/` из форков.

## 4. Взаимодействие узлов
### 4.1 Сигнал связи
Когда участник А хочет связаться с участником Б, система создаёт в репозитории А файл `signals/Б.yml`, а в репозитории Б открывает Issue с предложением связи. При подтверждении появляются взаимные ссылки.

### 4.2 Индексация
Портал периодически сканирует форки, считывает `config.yml` и `signals/`, строит карту сети (Cytoscape.js).

## 5. Технологический стек
| Слой | Технология |
|------|------------|
| Портал | Next.js 15 + Tailwind + shadcn/ui |
| Десктопный узел | Tauri (для оффлайн-работы) |
| ИИ-интерпретатор | Ollama / LM Studio + Llama 3.1 или Grok API |
| Визуализация | Cytoscape.js |

## 6. Дорожная карта
- **Фаза 0 (текущая):** Создание репозитория, документации, шаблона узла.
- **Фаза 1:** Прототип «Петли» на портале, авторизация через GitHub, создание форка.
- **Фаза 2:** Механизм сигналов связи, карта сети.
- **Фаза 3:** Локальные LLM, оффлайн-работа.
